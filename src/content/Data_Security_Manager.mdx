## CIA Triad

> 정보 보안의 가장 기본적이고 핵심적인 3가지 원칙

- 기밀성
- 무결성
- 가용성

### 기밀성(Confidentiality)

**-** 승인된 사람만이 정보에 접근할 수 있도록 보장하는 것. <br/>

##### 위협

**-** 정보 유출, 스니핑(Sniffing), 권한 없는 접근 등

<br />

### 무결성(Integrity)

**-** 정보가 정확하고 완전하며, 허가되지 않은 방식으로 훼손, 변조, 삭제되지 않고 원본 그대로의 상태를 유지하고 있음을 보장하는 것<br/>

##### 위협

**-** 데이터 변조, 위조 등

<br />

### 가용성(Availability)

**-** 권한이 있는 사용자가 필요할 때 언제든지 정보 및 시스템에 접근하여 사용할 수 있도록 보장하는 것<br/>

##### 위협

**-** 서비스 거부(DoS/DDoS) 공격 등

## 암호학의 기초

- 대칭키
- 비대칭키
- 해시함수

### 대칭키

> 데이터를 암호화하고 복호화할 때 **동일한 하나의 키(비밀 키)** 를 사용하는 암호화 방식

##### 작동 방식

1. 비밀 키 공유: 통신하려는 두 사람이 동일한 비밀 키를 안전하게 미리 공유한다.
2. 데이터 암호화 (송신자): 송신자가 이 비밀 키를 사용해 원본 데이터를 암호문으로 만든다.
3. 데이터 복호화 (수신자): 수신자가 동일한 비밀 키를 사용해 암호문을 다시 원본 데이터로 복원한다.

##### 특징

- 빠르다.
- 구조가 비교적 간단하다.
- 키 관리가 어렵다.

##### 예시 알고리즘

**-** AES, DES 등

<br />

### 비대칭키

> 데이터를 암호화하고 복호화할 때 **서로 다른 한 쌍의 키(공개키, 개인키)** 를 사용하는 암호화 방식

##### 작동 방식

**-** **비대칭키 암호화 작동 방식**

1. 공개 키 배포: 수신자가 자신의 공개 키를 송신자에게 전달한다.
2. 데이터 암호화 (송신자): 송신자가 수신자의 공개 키로 데이터를 암호화한다.
3. 데이터 복호화 (수신자): 수신자가 자신의 개인 키로 암호문을 복호화해 원본 데이터를 얻는다.

**-** **전자서명 작동 방식**

1. 해시 생성 (송신자): 송신자가 메시지의 해시값을 계산한다.
2. 서명 생성 (송신자): 해시값을 자신의 개인 키로 암호화해 전자서명을 만든다.
3. 서명 및 메시지 전송: 서명과 원본 메시지를 함께 보낸다.
4. 서명 검증 (수신자): 수신자가 송신자의 공개 키로 서명을 복호화하고, 메시지의 해시값을 다시 계산해 비교하여 위조 여부와 발신자를 확인한다.

##### 특징

- 속도가 느리다.
- 키 분배가 용이하다.
- 인증 및 부인 방지 기능 제공.

##### 예시 알고리즘

**-** RSA 등

<br />

### 해시 함수

> 임의의 길이를 갖는 데이터를 입력받아 고정된 길이의 데이터 (해시 값) 로 변환하는 함수, 데이터의 무결성을 검증하는데 사용된다.

##### 해시 함수의 활용 예시

- **데이터 무결성 검증**: 파일을 다운로드하거나 중요한 데이터를 전송할 때, 데이터가 손상되거나 변조되지 않았는지 확인하는 데 사용된다.
- **비밀번호 저장**: 사용자 비밀번호를 해시 함수로 변환하여 저장되므로, 데이터베이스가 유출되더라도 실제 비밀번호는 노출되지 않아 보안을 강화한다.
- **블록체인**: 데이터의 무결성, 보안, 식별성을 확보하기 위해 사용된다.

##### 암호학적 해시함수

**-** 특징: 역상(pre-image), 제2역상(2nd pre-image), 충돌쌍(collision)에 대하여 안전성을 가져야 하며, 인증에 이용된다.

**-** 종류: MD5, SHA계열

##### 비암호학적 해시함수

**-** 특징: 데이터 처리의 효율성과 속도에 초점을 맞춘, 암호학적인 보안 요구사항이 없는 환경에서 주로 사용된다.

**-** 종류: CRC32등

## 각종 인증 및 인가 메커니즘

- PKI

- OpenID

### PKI (Public Key Infrastructure)

> 공개 키 암호화 기술을 기반으로 하는 보안 시스템, 디지털 환경에서 사용자, 장치, 서비스의 신원을 확인하고 안전하게 통신할 수 있도록 해주는 일련의 구성 요소(하드웨어, 소프트웨어, 정책, 절차)를 총칭한다.

##### PKI의 주요 기능

- 기밀성
- 인증
- 무결성
- 부인방지

##### PKI 사용 예시

- 웹 브라우징 보안(HTTPS / SSL/TLS): PKI 기반의 SSL/TLS프로토콜을 사용해 통신을 암호화 하고 서버의 신원을 인증한다.
- 전자 서명 및 본인 인증 (공동인증서, 민간인증서): 사용자는 자신의 개인 키를 사용하여 전자 문서에 서명하고, 그 서명의 진위 여부는 사용자의 공개 키와 CA가 발급한 디지털 인증서를 통해 검증된다.

### OAuth (Open Authorization)

> 어플리케이션을 이용할 때 사용자가 해당 어플리케이션에 ID, PW등의 정보를 제공하지 않고, 신뢰할 수 있는 외부 어플리케이션(Naver, Google, Kakao, Facebook 등)의 Open API에 ID, PW를 입력하여 **해당 어플리케이션이 인증 과정을 처리해주는 방식**.

##### OAuth를 사용하는 이유

- 보안
- 편의성

<br />
### OpenID (Open IDentity)

> 사용자가 여러 웹사이트에서 하나의 ID로 로그인할 수 있게 해주는 인증 방식

##### OpenID를 사용하는 이유

- 매번 새로운 계정 만들 필요 없이 하나의 계정으로 여러 사이트에 로그인 하기 위함

## 접근 제어 정책

- MAC
- DAC
- RBAC

### MAC (Mandatory Access Control, 강제적 접근 제어)

> 권한 할당을 정책과 규칙에 따라 강제하는 접근 제어 모델<br/>
> 주체(사용자)와 객체(자원)에 대한 레이블(label)이 할당되며, 보안 정책에 따라 레이블 간의 비교를 통해 접근 권한을 결정한다.

### DAC (Discretionary Access Control, 임의적 접근 제어)

> 주체(사용자)가 객체(자원)에 대한 접근 권한을 결정하는 접근 제어 모델<br/>
> 이 모델에서는 객체에 대한 소유자가 해당 객체에 대한 접근 권한을 설정할 수 있으며, 소유자는 권한을 다른 사용자에게 부여하거나 취소할 수 있다.

### RBAC (Role-based Access Control, 역할 기반 접근 제어)

> 사용자의 역할을 기반으로 접근 권한을 제어하는 접근 제어 모델
> 이 모델에서는 역할(role)을 정의하고, 각 역할에 대해 해당 역할이 수행할 수 있는 작업에 대한 권한을 할당한다. 사용자는 하나 이상의 역할을 가질 수 있으며, 사용자가 속한 역할에 따라 접근 권한이 결정된다.

## 네트워크 보안

- 방화벽
- VPN
- NAC

<br />

### 방화벽

> 미리 정의된 보안 규칙에 기반한, 들어오고 나가는 네트워크 트래픽을 모니터링하고 제어하는 네트워크 보안 시스템

##### 방화벽의 주요 기능

- 내부 네트워크와 외부 네트워크 간의 접근 제어
- 서비스 포트 제어
- 네트워크 주소 변환(NAT)
- 로깅 및 감사

<br />

### VPN (Virtual Private Network, 가상 사설 네트워크)

> 공용 네트워크(인터넷)를 통해서도 마치 자신만의 사설 네트워크처럼 안전하고 암호화된 통신을 가능하게 해주는 기술

##### VPN 사용 이유

- 해외 콘텐츠 이용 (넷플릭스, 유튜브 지역 제한 우회)
- 보안 강화
- 회사 내부망 접속 (재택근무 시 자주 사용)
- 검열 우회 (특정 국가에서 차단된 사이트 접속)

<br />

### NAC (Network Access Control)

> 엔드포인트(PC, 스마트폰 등과 같은 네트워크에 접속하는 모든 유무선 기기)가 네트워크에 접근하기 전 보안 정책 준수 여부를 검사하여 네트워크 사용을 제어하는 것

## 보안 사고 대응 프로세스 (Incident Response)

> 조직이 사이버 보안 침해 또는 공격이 발생했을 때 이를 체계적으로 관리하고 해결하여 피해를 최소화하며, 재발을 방지하기 위한 일련의 절차와 전략

![침해사고대응](/img/Incident_Response.png)

### 1. 준비

> 공격이 발생하기 전에 모든 준비를 마치고 공격자를 한발 앞서는 과정

##### 기술적 측면:

- 원격측정: 침입을 확인하고 조사하는 데 필요한 시스템을 미리 구축하여 다양한 활동을 관찰하고 조사할 수 있도록 한다.
- 경화: 침입을 막기 위해 시스템 패치, 설정 확인, VPN 및 방화벽과 같은 공격 제한 도구 설치 등 기본적인 보안 강화 조치를 취한다.

##### 비기술적 측면:

- 절차 및 문서화: 사고 대응 매뉴얼을 사전에 마련하고 관련 내용을 문서화하여 사고 발생 시 신속하게 대응할 수 있도록 한다.
- 연습: 매뉴얼에 따라 신속하게 대응하기 위해 모의 훈련을 진행하며, 필요시 레드팀을 섭외하여 실제 사이버 공격 훈련을 감행한다.

### 2. 확인 단계

> 공격자의 침입을 감지하는 과정. 네트워크 트래픽 분석, 감염된 호스트의 명령 및 제어 트래픽 인식, 대규모 데이터 추출 트래픽 확인 등을 통해 침입을 인지 한다.

### 3. 봉쇄 단계

> 공격자의 행위를 완화하기 위한 첫 번째 능동적인 조치이다.

- 네트워크 포트 비활성화
- 방화벽 ACL 추가(IP 주소, 도메인, URL 차단)
- 침입자가 제어하는 사용자 계정 잠금
  <br />
  등등..

### 4. 박멸 단계

> 공격자가 시스템에 다시 접근하지 못하도록 장기적인 조치를 수행한다.

### 5. 복구 단계

> 시스템을 공격 이전의 정상 상태로 되돌리는 사후 처리 조치를 한다.

### 6. 교훈 단계

> 사고 대응 이후 프로세스 및 시스템 개선을 위한 방안을 도출한다.

## 취약점 스캐닝 및 펜테스트 기초

- 취약점 스캐닝
- 펜테스트

### 취약점 스캐닝

> 자동화된 도구를 사용하여 시스템, 애플리케이션 또는 네트워크에서 알려진 보안 취약점을 식별하는 과정

##### 취약점 스캐닝의 특징

- **자동화**
- **빠른 속도**
- **알려진 취약점 식별**: 최신 보안 데이터베이스를 활용해 이미 알려진 보안 문제를 찾아낸다.
- **넓은 범위의 커버리지**: 특정 컴퓨터만 보는 게 아니라 네트워크 전체, 다양한 운영체제, 애플리케이션, 데이터베이스 등 넓은 범위를 확인한다.
- **정기적 수행 용이성**
- **초기 단계 보안 강화**

<br />

### 펜테스트(Penetration Test)

> 실제 공격자가 어떤 방식으로 침투할 수 있는지 모의로 공격해보고 그 과정을 검증하는 것 (모의 해킹)

##### 펜테스트의 순서

1. 계획 및 정찰
2. 취약점 분석
3. 공격 및 침투
4. 권한 상승 및 추가 공격
5. 흔적 제거 및 보고

### 취약점 테스트와 펜테스트의 차이

> 취약점 스캐닝은 알려진 취약점을 이라면 빠르게 탐지 가능. <br/>
> 펜테스트는 취약점 스캐닝으로 찾지 못한 취약점 발견 가능.
